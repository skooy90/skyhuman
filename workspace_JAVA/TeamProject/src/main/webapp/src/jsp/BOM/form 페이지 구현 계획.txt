BOM_form.jsp 구현 계획

1. 페이지 개요
- BOM 등록/수정 페이지
- URL 파라미터로 등록/수정 모드 구분 (?code=B0001)
- 등록시: BOM번호 자동생성, 모든 필드 입력 가능
- 수정시: BOM번호 readonly, 기존 데이터 표시

2. 입력 필드
2.1 기본 정보
- BOM번호: 등록시 자동 생성 (B0001, B0002...), 수정시 readonly
- 제품코드: 드롭다운 (StandardDTO에서 선택)
- 제품명: 제품코드 선택시 자동 표시 (readonly)
- BOM 설명: 텍스트 입력 (예: "오이 천연비누 BOM")
- BOM 유형: 드롭다운 (원자재, 반제품, 완제품)
- BOM 순서: 숫자 입력 (1, 2, 3...)

2.2 자재 정보 (동적 추가/삭제)
- 자재코드: 드롭다운 (MaterialDTO에서 선택)
- 자재명: 자재코드 선택시 자동 표시 (readonly)
- 필요수량: 숫자 입력 (소수점 가능)
- 단위: 자재 선택시 자동 표시 (readonly)
- 비고: 텍스트 입력 (선택사항)

3. 유효성 검사
3.1 클라이언트 사이드
- 필수 필드: 제품코드, BOM 설명, BOM 유형, BOM 순서
- BOM 순서: 1 이상의 정수
- BOM 유형: 선택된 값만 허용
- 자재 정보: 최소 1개 이상 필요
- 필요수량: 0보다 큰 숫자

3.2 서버 사이드
- BOM번호 중복 체크 (등록시)
- 제품코드 존재 여부 확인
- 자재코드 존재 여부 확인
- BOM 순서 중복 체크 (같은 제품 내)

4. UI/UX 기능
4.1 자재 추가/삭제
- "자재 추가" 버튼으로 새 자재 행 추가
- 각 자재 행에 "삭제" 버튼
- 최소 1개 자재는 유지

4.2 자동 완성
- 제품코드 선택시 제품명 자동 표시
- 자재코드 선택시 자재명, 단위 자동 표시

4.3 폼 모드별 표시
- 등록 모드: "BOM 등록" 제목, "등록" 버튼
- 수정 모드: "BOM 수정" 제목, "수정" 버튼

5. 샘플 데이터 (하드코딩)
5.1 제품 목록 (StandardDTO)
- RA001: 신선 오이
- SE001: 오이 비누 베이스
- FI001: 오이 천연비누
- FI002: 오이 각질제거비누
- FI003: 오이 수축비누

5.2 자재 목록 (MaterialDTO)
- MA0001: 신선 오이 (kg)
- MA0002: 천연 비누 베이스 (kg)
- MA0003: 오이 추출물 (ml)
- MA0004: 천연 보습제 (ml)
- MA0005: 천연 각질제거제 (g)
- MA0006: 천연 수축제 (ml)

5.3 수정 모드 샘플 데이터
- BOM번호: B0001
- 제품코드: FI001 (오이 천연비누)
- BOM 설명: 오이 천연비누 BOM
- BOM 유형: 완제품
- BOM 순서: 1
- 자재 정보:
  * MA0002: 천연 비누 베이스, 1.0kg
  * MA0003: 오이 추출물, 50ml
  * MA0004: 천연 보습제, 30ml

6. 페이지 구조
6.1 헤더
- 페이지 제목 (등록/수정)
- 뒤로가기 버튼

6.2 폼 섹션
- 기본 정보 입력 영역
- 자재 정보 입력 영역 (동적 테이블)

6.3 버튼 영역
- 등록/수정 버튼
- 취소 버튼

7. JavaScript 기능
7.1 폼 검증
- 실시간 유효성 검사
- 에러 메시지 표시

7.2 동적 자재 관리
- 자재 행 추가/삭제
- 자재 정보 자동 완성

7.3 폼 모드 처리
- URL 파라미터로 모드 구분
- 모드별 필드 상태 변경
